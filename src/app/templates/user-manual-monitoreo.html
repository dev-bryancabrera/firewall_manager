Monitoreo de Trafico de Red¶

El monitoreo de tráfico de red es una práctica que implica la supervisión continua del flujo de datos a través de una red. Su objetivo principal es garantizar el rendimiento, la seguridad y la eficiencia de la red, permitiendo identificar problemas como congestión, amenazas de seguridad y fallos en el sistema. Mediante el uso de herramientas especializadas, los administradores de red pueden analizar patrones de tráfico, detectar anomalías y tomar decisiones informadas para optimizar el funcionamiento de la infraestructura de red.

Paquetes de Red ¶

Los paquetes de red son pequeñas unidades de datos que se envían a través de una red. Cada paquete incluye tanto la información que se quiere transmitir como las direcciones del remitente y el destinatario.

¿Qué es un paquete de red? ¶

Los paquetes de red son unidades fundamentales de datos que se utilizan para enviar información a través de redes digitales, incluyendo Internet. Cuando se envían datos desde un dispositivo a otro, estos se dividen en segmentos más pequeños llamados paquetes. Cada paquete contiene varios elementos clave:

  • Dirección de Origen: La dirección IP del dispositivo que envía el paquete.
  • Dirección de Destino: La dirección IP del dispositivo que debe recibir el paquete.
  • Número de Secuencia: Indica el orden de los paquetes para que el receptor pueda reensamblarlos correctamente.
  • Protocolo: El protocolo de comunicación utilizado, como TCP o UDP.
  • Flags: Bits que controlan aspectos como el establecimiento de la conexión (SYN, ACK, FIN).
  • TTL (Time to Live): Evita que los paquetes circulen indefinidamente, siendo decrementado en cada nodo hasta que el paquete se descarta.

¿Cómo funcionan los paquetes de red? ¶

Los paquetes de red son esenciales para la comunicación digital, permitiendo el intercambio eficiente y fiable de datos en redes complejas. Comprender cómo funcionan y cómo se gestionan es crucial para mantener la seguridad, el rendimiento y la integridad de las redes modernas.

  • Fragmentación: Cuando los datos que se van a enviar son mayores que el tamaño máximo permitido para un solo paquete, se dividen en múltiples paquetes. Cada uno de estos paquetes contiene parte de los datos y la información necesaria para que el receptor pueda reensamblarlos correctamente.
  • Encaminamiento (Routing): Los paquetes pueden tomar diferentes rutas para llegar a su destino. Los routers y switches en la red utilizan tablas de encaminamiento y algoritmos para determinar la mejor ruta para cada paquete, considerando factores como la congestión de la red y las fallas en los enlaces.
  • Control de Flujo y Congestión: En el caso de TCP, existen mecanismos para ajustar la velocidad de transmisión de los paquetes en función de la capacidad del receptor y la carga actual de la red, lo que ayuda a evitar la congestión y la pérdida de paquetes.
  • Reensamblaje: En el dispositivo de destino, los paquetes se reensamblan en su orden original utilizando los números de secuencia. Si algún paquete falta o está corrupto, se solicita su retransmisión.

Configuración de filtros de captura ¶

En el menú principal, una de las opciones es el Monitoreo de tráfico de red. Al ingresar, como ventana principal, veremos una tabla vacía donde se mostrarán los filtros que creemos.

1. Crear nuevo filtro ¶

La creación de filtros nos ayuda a definir cómo y qué tipo de tráfico queremos capturar. Podemos personalizar el tráfico a seguir, ya sea HTTP, HTTPS, SSH, entre otros.

Para crear un filtro correctamente, debemos tener claro el tipo de tráfico que queremos capturar, ya sea dominios, contenidos o tráfico del servidor como SSH, MySQL, etc. Debemos considerar lo siguiente:

  1. Presionamos el botón Crear filtro y se desplegará un formulario que debemos llenar según la explicación de cada campo.
  2. En el formulario, configuramos y completamos con la información correcta teniendo en cuenta los siguientes puntos:
    • Campo Nombre del Filtro: Definimos el nombre del filtro como un alias para identificarlo.
    • Campo Tipo de Filtro: Escogemos el tipo de tráfico a monitorear: contenido, dominios, IP, puerto, protocolo de red, etc.

      Nota

      Al seleccionar un tipo de filtro específico, se habilitarán ciertos campos según la elección.

      • Tipo de regla - Contenido
        • Campo Filtro de Contenido: Escogemos si queremos capturar únicamente paquetes que establezcan comunicación hacia un dominio dentro del tipo de contenido (Contenido de Destino) o si queremos visualizar todos los paquetes que salen del servidor (Contenido General).
        • Campo Contenido de Destino o Campo Contenido General: Estos campos cambian según la selección anterior. Escogemos de la lista el contenido a filtrar.
      • Tipo de regla - Dominio
        • Campo Filtro de Dominio: Similar a la elección anterior, elegimos si queremos capturar paquetes hacia un dominio específico (Dominio de Destino) o todos los paquetes que salen del servidor (Dominio General).
        • Campo Dominio de Destino o Campo Dominio General: Estos campos cambian según la selección anterior. Ingresamos el dominio a filtrar.
      • Tipo de regla - Personalizar IP y Puerto

          Al seleccionar esta opción, los campos funcionan de manera similar. Explicamos uno para evitar redundancias.

        • Campos Filtro de IPs, Filtro de Puertos, Filtro de Protocolos de Red: Podemos especificar si el filtro capturará paquetes solo de origen, solo de destino, general (origen y destino), o establecer un filtro con datos de origen y destino, por ejemplo, tráfico de la IP 192.168.0.100 a la IP 192.168.0.111.
        • Campo IP de origen, Puerto de origen, Protocolo de red de origen: Indicamos el valor de origen (puerto, IP, protocolo de red).
        • Campo IP de destino, Puerto de destino, Protocolo de red de destino: Indicamos el valor de destino (puerto, IP, protocolo de red).
        • Campo IP general, Puerto general, Protocolo general: Indicamos el valor general para filtrar el tráfico saliente o entrante.
        • Campo Protocolo: Filtramos el tráfico por tipo de protocolo de comunicación, TCP o UDP.
      Hay campos adicionales que se habilitan o deshabilitan según cómo llenemos los campos. En el caso de las IPs, si especificamos origen y destino, aparecerá un campo adicional para definir si se deben cumplir ambos requisitos o uno de ellos.
      • Campo ¿Cómo deseas filtrar?: Indicamos si queremos cumplir ambos criterios o uno de ellos para filtrar los paquetes.
  3. Una vez entendidos los conceptos de cada campo, definimos el filtro, ingresamos los valores y presionamos el botón Crear Filtro.
  4. Para verificar que la regla se creó correctamente, debe aparecer un mensaje de confirmación y podremos ver el filtro creado en la tabla.

2. Revisar reporte ¶

Dentro de la ventana de los filtros, encontramos el botón Revisar reporte, disponible una vez guardado algún reporte de un filtro creado.

Al presionar este botón, nos redirige a otra página que se explica más adelante.

3. Capturar tráfico ¶

Al crear un filtro, en la celda del registro tendremos el botón Capturar tráfico, que nos permite aplicar el filtro para capturar paquetes. Al igual que el botón anterior, nos redirige a otra página explicada más adelante.

Es importante recalcar que cada filtro tiene adicionalmente un botón para aplicar el filtro y capturar paquetes.

4. Limpiar tráfico capturado ¶

Otra opción disponible al crear un registro de un nuevo filtro, después de capturar paquetes, es el botón Limpiar tráfico capturado. Este botón permite limpiar una captura anterior para aplicar un nuevo filtro y evitar confusiones.

5. Eliminar filtro ¶

La última opción en esta página es eliminar un filtro creado, ya sea porque no lo vamos a usar o queremos crear uno más completo. Ubicamos el botón Eliminar filtro que abre un mensaje de confirmación.

Para comprobar que la acción se realizó con éxito, debe aparecer un mensaje de Filtro eliminado correctamente y los registros actualizados en la tabla.

Monitoreo de paquetes ¶

Cuando aplicamos un filtro para capturar el tráfico o queremos revisar algún reporte, se nos presentará una nueva página con una tabla vacía y botones que realizan diferentes acciones según lo que deseemos hacer. A continuación, se explican las acciones que se pueden realizar en esta página.

Nota

Es importante tener en cuenta que en la página para ver el tráfico de paquetes, encontraremos dos tipos de acciones. Por ejemplo, al capturar tráfico con algún filtro, veremos botones para interactuar con el tráfico, como capturar paquetes o guardar reportes. Pero al revisar un reporte específico, habrá otros botones para cargar un reporte guardado, sin relación con algún filtro.

1. Limpiar tabla ¶

Una de las acciones que podemos realizar en esta ventana es Limpiar tabla. Este botón nos permite borrar los datos de la tabla para evitar la sobrecarga de la página o eliminar paquetes de un filtro anterior, de modo que podamos comenzar una nueva captura de paquetes.

2. Botón de ayuda ¶

Otra opción disponible es el botón informativo, que abre una tarjeta con información sobre la captura del paquete, como el Tipo de Servicio (TOS), Tiempo de Vida (TTL), Identificación (ID), Desplazamiento (Offset), Indicadores (Flags), Protocolo y Longitud.

3. Botón play/pause para la captura de paquetes ¶

Este botón nos permite iniciar o pausar la captura de paquetes. Al ingresar a esta ventana, el botón Capturar paquetes abrirá la página, pero no comenzará a capturar hasta que presionemos Play. Para pausar la captura, presionamos nuevamente el botón, y para reanudar, volvemos a presionar Play.

Nota

Al comenzar a capturar paquetes, la información no se mostrará instantáneamente ya que debe cumplir los requisitos del filtro. Si no vemos información, podría ser porque el filtro no está configurado correctamente.

4. Guardar reporte ¶

Este botón nos permite guardar un reporte generado al capturar paquetes. Debemos pausar la captura antes de guardar, y al presionar el botón, se nos pedirá asignar un nombre al reporte. Al confirmar, se mostrará un mensaje de Reporte guardado correctamente, indicando que el reporte está disponible para visualizar en el futuro.

5. Cargar reporte ¶

Al guardar un reporte, este botón abrirá una ventana con una lista de reportes guardados. Tenemos dos opciones: Mostrar datos para cargar el reporte guardado, y Eliminar reporte para eliminar un reporte que ya no necesitemos.

6. Regresar a los Filtros ¶

Por último, el botón Regresar a los Filtros nos permite volver a la ventana con la tabla de filtros, para aplicar otro filtro o realizar una nueva captura de paquetes.
